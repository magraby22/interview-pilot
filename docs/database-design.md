# Database Design
## Overview

The InterviewPilot database is designed to support AI-powered technical interviews.

The current design focuses on the Minimum Viable Product (MVP), providing a simple, maintainable, and scalable relational database structure.

## Entities

## User

**Purpose**
Represents a registered user of the InterviewPilot platform.
### Attributes
| Attribute | Data Type | Description |
|-----------|-----------|-------------|
| id | BIGSERIAL | Unique identifier for each user. |
| first_name | VARCHAR(50) | User's first name. |
| last_name | VARCHAR(50) | User's last name. |
| email | VARCHAR(255) | User's unique email address. |
| password_hash | TEXT | Hashed password. |
| created_at | TIMESTAMP | Account creation timestamp. |
| updated_at | TIMESTAMP | Last update timestamp. |

### Relationships
- A **User** can have **many Interview Sessions**.
- Each **Interview Session** belongs to **one User**.

## InterviewSession

### Purpose

Represents a single interview attempt created by a user. It stores the interview metadata and acts as the parent entity for generated questions and the final AI evaluation.

### Attributes

| Attribute | Data Type | Description |
|-----------|-----------|-------------|
| id | BIGSERIAL | Unique identifier for each interview session. |
| user_id | BIGINT | References the owner of the interview session. |
| position | TEXT | Target job position (e.g., Backend, Frontend, Full Stack). |
| level | TEXT | Target experience level (e.g., Junior, Mid-Level, Senior). |
| status | TEXT | Current interview status (IN_PROGRESS, COMPLETED, CANCELLED). |
| started_at | TIMESTAMP | Interview start time. |
| ended_at | TIMESTAMP | Interview finish time. |

### Relationships

- Belongs to **one User**.
- Contains **many Questions**.
- Has **one Evaluation**.

## Question

### Purpose

Represents a technical interview question generated dynamically by the AI during an interview session. Each question belongs to a single interview session and is presented to the user in a specific order.

### Attributes

| Attribute | Data Type | Description |
|-----------|-----------|-------------|
| id | BIGSERIAL | Unique identifier for each question. |
| session_id | BIGINT | References the interview session. |
| question_text | TEXT | The question generated by the AI. |
| question_order | INTEGER | The order of the question within the interview session. |

### Relationships

- Belongs to **one Interview Session**.
- Can have **zero or one Answer**.

## Answer

### Purpose

Represents the user's response to a generated interview question. An answer is optional because a user may skip a question or leave the interview before answering.

### Attributes

| Attribute | Data Type | Description |
|-----------|-----------|-------------|
| id | BIGSERIAL | Unique identifier for each answer. |
| question_id | BIGINT | References the corresponding question. |
| answer_text | TEXT | The user's answer. |

### Relationships

- Belongs to **one Question**.

## Evaluation

### Purpose

Represents the overall AI evaluation of an interview session. Instead of evaluating each question individually, the platform generates a single comprehensive assessment after the interview is completed.

### Attributes

| Attribute | Data Type | Description |
|-----------|-----------|-------------|
| id | BIGSERIAL | Unique identifier for each evaluation. |
| session_id | BIGINT | References the evaluated interview session. |
| technical_score | DECIMAL(3,1) | Overall technical performance score. |
| completeness_score | DECIMAL(3,1) | Measures how completely the user answered the interview questions. |
| strengths | JSONB | List of the candidate's strengths generated by the AI. |
| weaknesses | JSONB | List of the candidate's weaknesses generated by the AI. |
| recommendation | TEXT | Personalized recommendations for improvement generated by the AI. |

### Relationships

- Belongs to **one Interview Session**.

## Relationship Summary

The InterviewPilot database follows a relational design centered around the interview session.

- A **User** can have multiple **Interview Sessions**.
- Each **Interview Session** contains multiple **Questions**.
- Each **Question** can have zero or one **Answer**.
- Each **Interview Session** has one overall **Evaluation** generated by the AI after the interview is completed.
